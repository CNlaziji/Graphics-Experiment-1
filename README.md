# OpenGL 实验工程模板

## 项目概述

这是一个基于 OpenGL 3.3 Core Profile 的实验工程模板，旨在为学习者提供一个完整的 OpenGL 开发环境，包含了基本的窗口管理、着色器编译、相机控制和三维渲染功能。本模板适合用于计算机图形学课程的实验教学，以及 OpenGL 初学者的学习和练习。

## 项目特点

- **完整的依赖库**：包含完整的 GLAD 和 GLM 库，减少配置麻烦
- **彩色立方体**：使用不同颜色显示立方体的不同面，增强 3D 效果
- **相机控制**：支持 WASD 键控制相机移动，提供沉浸式体验
- **投影切换**：支持透视投影和正交投影的切换
- **模块化设计**：代码结构清晰，易于理解和扩展
- **完善的注释**：代码中包含详细的注释，便于学习和使用
- **跨平台兼容**：使用 CMake 构建系统，支持多种平台
- **丰富的实验任务**：提供了从二维到三维的多种实验任务

## 工程结构

```
.
├── CMakeLists.txt      # CMake 配置文件
├── main.cpp            # 主文件，包含渲染循环和核心逻辑
├── Shader.h            # 着色器类，封装着色器编译和使用
├── Shader.cpp          # 着色器类实现
├── Camera.h            # 相机类，封装相机控制和视图矩阵计算
├── glad/               # GLAD 库，用于 OpenGL 函数加载
│   ├── include/        # GLAD 头文件
│   └── src/            # GLAD 源文件
├── glm/                # 完整的 GLM 数学库，用于向量和矩阵运算
├── vcpkg/              # 依赖库管理（如果使用 vcpkg）
└── README.md           # 本说明文件
```

## 依赖库

- **GLFW**：用于窗口管理和输入处理
- **GLAD**：用于 OpenGL 函数动态加载
- **GLM**：用于数学计算（本模板包含完整的官方 GLM 0.9.9.8 版本）

## 快速开始

1. **克隆或下载项目**
   - 将项目复制到本地目录

2. **确保 GLFW 已安装**
   - 如果使用模板自带的依赖，将 GLFW 的 `include` 目录复制到 `vcpkg/glfw/include`
   - 如果使用 vcpkg，运行 `vcpkg install glfw3 glad glm`

3. **编译项目**
   ```bash
   mkdir build && cd build
   cmake ..
   cmake --build .
   ```

4. **运行项目**
   - 编译成功后，运行生成的可执行文件
   - 使用 WASD 键控制相机移动
   - 使用 P/O 键切换投影模式
   - 观察彩色立方体的 3D 效果

## 配置说明

### 1. 安装依赖库

本模板已经包含了大部分依赖库，你只需要确保 GLFW 已正确安装：

#### 方法一：使用模板自带的依赖

1. **GLAD**：模板已包含完整的 GLAD 库
2. **GLM**：模板已包含完整的官方 GLM 0.9.9.8 版本
3. **GLFW**：需要确保已正确安装
   - 下载地址：https://www.glfw.org/download.html
   - 解压后将 `include` 目录复制到 `vcpkg/glfw/include` 目录，或使用包管理器安装

#### 方法二：使用 vcpkg（推荐）

1. 安装 vcpkg：https://github.com/microsoft/vcpkg
2. 安装依赖：
   ```bash
   vcpkg install glfw3 glad glm
   ```
3. 在 CMakeLists.txt 中配置 vcpkg 路径

### 2. 编译项目

#### 使用 CMake

1. 创建 build 目录：
   ```bash
   mkdir build && cd build
   ```

2. 运行 CMake：
   ```bash
   cmake ..
   ```
   （如果使用 vcpkg，需要指定工具链文件：
   ```bash
   cmake .. -DCMAKE_TOOLCHAIN_FILE=[vcpkg路径]/scripts/buildsystems/vcpkg.cmake
   ```
   ）

3. 编译项目：
   ```bash
   cmake --build .
   ```

### 3. 运行项目

- 编译成功后，运行生成的可执行文件
- **控制说明**：
  - `W` 键：向前移动相机
  - `S` 键：向后移动相机
  - `A` 键：向左移动相机
  - `D` 键：向右移动相机
  - `P` 键：切换到透视投影
  - `O` 键：切换到正交投影
  - `ESC` 键：退出程序

- **运行效果**：
  - 显示一个彩色立方体，不同面显示不同颜色（红色、绿色、蓝色）
  - 支持相机移动，从不同角度观察立方体
  - 支持透视/正交投影切换
  - 立方体的 3D 结构清晰可见

## 实验任务点

### 1. 二维图形绘制
- **任务**：在 `Transform` 类的 `applyTransforms` 方法中实现二维图形绘制
- **实现提示**：可以通过修改顶点数据来绘制不同的二维图形，如三角形、矩形等

### 2. 二维变换
- **任务**：在 `Transform` 类的 `applyTransforms` 方法中实现以下变换：
  - 缩放（Scaling）
  - 反射（Reflection）
  - 切变（Shearing）
  - 旋转（Rotation）
  - 平移（Translation）
- **实现提示**：为每种变换创建对应的矩阵，并将它们应用到模型矩阵中

### 3. 二维组合变换
- **任务**：在 `Transform` 类的 `applyTransforms` 方法中实现二维组合变换
- **实现提示**：通过矩阵乘法将多个变换组合在一起，注意变换的顺序会影响结果

### 4. 三维图形绘制
- **任务**：在 `main.cpp` 中修改顶点数据实现三维图形绘制
- **实现提示**：可以绘制立方体、球体等基本三维几何体，需要添加适当的顶点数据和索引数据

### 5. 三维变换
- **任务**：在 `Transform` 类的 `applyTransforms` 方法中实现以下三维变换：
  - 缩放（Scaling）
  - 平移（Translation）
  - 绕 x 轴旋转
  - 绕 y 轴旋转
  - 绕 z 轴旋转
- **实现提示**：为每种变换创建对应的 4x4 变换矩阵

### 6. 三维组合变换
- **任务**：在 `Transform` 类的 `applyTransforms` 方法中实现三维组合变换
- **实现提示**：通过矩阵乘法将多个三维变换组合在一起

### 7. 键盘控制
- **任务**：在 `processInput` 函数中实现键盘控制
- **实现提示**：添加更多的键盘控制功能，如相机旋转、速度调整等

## 代码说明

### 核心文件

- **main.cpp**：
  - 包含主函数和渲染循环
  - 初始化 GLFW 和 GLAD
  - 创建窗口和上下文
  - 定义彩色立方体的顶点数据和缓冲区对象
  - 实现投影矩阵计算和切换（透视/正交）
  - 调用着色器和绘制函数
  - 处理用户输入和相机控制

- **Shader.h/Shader.cpp**：
  - 封装了着色器的编译、链接和使用
  - 提供了设置 uniform 变量的方法（布尔值、整数、浮点数、向量、矩阵）
  - 包含错误检查功能，确保着色器编译和链接成功

- **Camera.h**：
  - 封装了相机的基本功能
  - 实现了相机移动（WASD 控制）和方向向量计算
  - 提供了获取视图矩阵的方法，使用 GLM 的 lookAt 函数

- **Transform 类**：
  - 定义在 main.cpp 中
  - 管理模型矩阵
  - 预留了变换逻辑的实现区域，包括二维和三维变换

### 关键概念

- **着色器**：使用 GLSL 编写的顶点着色器和片段着色器，负责渲染管线的不同阶段
- **顶点数据**：定义了绘制对象的几何形状
- **缓冲区对象**：用于存储顶点数据和索引数据，提高渲染性能
- **矩阵变换**：使用矩阵来实现模型的平移、旋转和缩放
- **相机系统**：模拟真实相机的视角和移动
- **投影矩阵**：将三维坐标转换为二维屏幕坐标

## 注意事项

1. **OpenGL 版本**：本模板使用 OpenGL 3.3 Core Profile，确保你的显卡支持该版本

2. **依赖库路径**：确保依赖库的路径设置正确，特别是 GLFW 和 GLAD 的包含路径

3. **编译环境**：推荐使用支持 C++11 或更高版本的编译器

4. **性能优化**：对于复杂场景，可以考虑使用索引缓冲区、顶点数组对象等优化手段

5. **错误处理**：代码中包含了基本的错误检查，但在实际应用中可能需要更详细的错误处理

## 故障排除

### 常见问题

1. **编译错误**：
   - 检查依赖库是否正确安装
   - 检查 CMake 配置是否正确
   - 检查编译器是否支持 C++11 或更高版本

2. **链接错误**：
   - 检查是否链接了正确的库文件
   - 检查 GLFW 和 GLAD 的库路径是否设置正确

3. **运行错误**：
   - 检查显卡是否支持 OpenGL 3.3
   - 检查窗口创建是否成功
   - 检查着色器编译是否成功

4. **渲染问题**：
   - 检查顶点数据是否正确
   - 检查矩阵变换是否正确
   - 检查着色器代码是否正确

### 调试技巧

1. **启用调试输出**：在代码中添加适当的日志输出，以便了解程序的执行流程

2. **使用调试器**：使用 IDE 的调试功能，逐步执行代码并检查变量值

3. **检查 OpenGL 错误**：在关键操作后调用 `glGetError()` 检查 OpenGL 错误

4. **简化场景**：如果遇到复杂问题，可以先简化场景，逐步添加功能，以便定位问题
